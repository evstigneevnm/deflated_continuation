5 For maximum compatibility with existing Fortran environments, the cuBLAS library uses column-major storage,
6 and 1-based indexing.
7 Since C and C++ use row-major storage, applications written in these languages can not use the native array
8 semantics for two-dimensional arrays. Instead, macros or inline functions should be defined to implement
9 matrices on top of one-dimensional arrays. For Fortran code ported to C in mechanical fashion, one may chose
10 to retain 1-based indexing to avoid the need to transform loops. In this case, the array index of a matrix
11 element in row ''i'' and column ''j'' can be computed via the following macro:
12 
13 #define IDX2F(i,j,ld) ((((j)-1)*(ld))+((i)-1))
14 
15 Here, ld refers to the leading dimension of the matrix, which in the case of column-major storage is the
16 number of rows of the allocated matrix (even if only a submatrix of it is being used).
17 For natively written C and C++ code, one would most likely choose 0-based indexing, in which case the array
18 index of a matrix element in row ''i'' and column 'j'' can be computed via the following macro:
19 
20 #define IDX2C(i,j,ld) (((j)*(ld))+(i))
21
